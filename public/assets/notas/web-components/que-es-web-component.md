# ✔ ¿qué son los Web Components?

**Son un estándar nativo que permite crear componentes reutilizables sin ningún framework.**

En otras palabras:

- React = runtime + JSX + librería
    
- Angular = framework completo
    
- Web Components = APIs del navegador
    

---

# ✔ Ventajas principales

- Se ejecutan en cualquier navegador moderno
    
- No requieren React/Angular/Vue
    
- Se integran en cualquier proyecto
    
- Encapsulan estilos
    
- Reutilizables entre frameworks
    

---

# ✔ Desventajas reales

- Son bastante “low level” (mucho boilerplate)
    
- No tienen router, estado, SSR, etc.
    
- Mayor curva de aprendizaje que React o Vue
    
- Difícil manejar estados complejos sin un framework adicional
    

Por eso muchas empresas los usan para:

✔ librerías de UI  
✔ microfrontends  
✔ widgets externos  
✔ extensiones  
✔ componentes compartidos

Pero no tanto para apps completas (aunque se puede).

---

# ✔ Explicación corta

> Web Components = **un conjunto de APIs nativas** (Custom Elements + Shadow DOM + Templates) que permiten crear componentes HTML personalizados, encapsulados y reutilizables.
